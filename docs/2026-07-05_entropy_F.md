## Why it matters (the problem it solves)

To split data well (a decision tree) or to score a classifier's confidence, you need one number for
**"how mixed / how uncertain is this group?"** Entropy is that number.

## Intuition — entropy = the SUSPENSE in a bucket

> [!tip] One question: *"how mixed is this bucket?"* Pure → no suspense. 50/50 → maximum suspense.

🔧 **SSD grounding — two bins of flash blocks** (● = good block, ○ = bad):
```
   BIN A (pure)          BIN B (50/50 mess)      BIN C (90/10)
   ┌──────────────┐      ┌──────────────┐        ┌──────────────┐
   │ ●●●●●●●●●●    │      │ ●●●●●○○○○○    │        │ ●●●●●●●●●○    │
   │ ●●●●●●●●●●    │      │ ○○●●○●○●○●    │        │ ●●●●●●●●●●    │
   └──────────────┘      └──────────────┘        └──────────────┘
   pull one blind →      pull one blind →         pull one blind →
   "obviously a ●, yawn" "no idea... 🥁"          "one rebel..."

   SUSPENSE METER  (this IS entropy):
   0 ├█░░░░░░░░░░┤ 1      0 ├██████████┤ 1        0 ├████?????░┤ 1
      A: entropy = 0         B: entropy = 1.0 (max)   C: entropy = ?  (PARKED)
```
- **Bin A** — 100 good, 0 bad. Pull blind → zero suspense, you already know it's a ●. **Entropy = 0.**
- **Bin B** — 50/50. Pull blind → a coin flip, maximum drum-roll. **Entropy = maximum (= 1 bit).**
- Everything else (like Bin C) sits between. The formula (`−Σ p·log₂p`, still owed) is just the
  *needle's mechanism* — but the FEELING (suspense = mixedness) comes first.
