Entropy — from Expected Value to Information Gain

s4 prep · ensembles · grade ★★★ rebuild-from-scratch
The thread
One machine, four concepts 🧵

Everything here is the same weighted-average loop wearing different hats. Read it as one chain:

EXPECTED VALUE → the loop itself (average a value, weighted by chance) ↓ feed it "surprise" instead of a plain value SURPRISE → how shocking one outcome is = −log₂(p) ↓ average the surprise over all outcomes ENTROPY → expected surprise = how mixed / uncertain a pile is ↓ measure how much a question shrinks it INFORMATION GAIN → entropy(before) − entropy(after) = how good a split is

Two example worlds carry it: 🐕 a dog-adoption shelter + 🎰 a betting table for the expected-value idea, then 🔧 bins of good/bad flash blocks for entropy & the tree split.

🎯 Why today: tomorrow's class is ensembles (random forest / XGBoost) — stacks of decision trees, and every tree splits by chasing information gain. This is the exact model family in your thermal-throttle research papers. Own this → the whole session is downhill.
Step 1 · Expected Value
Expected value = the long-run average, weighted by "how often"

If you repeat something forever, what do you get per try on average? Each outcome pulls the average toward itself, and its pull-strength is its probability. It's the balance-point of everything that could happen.

🐕 The shelter. A dog lands in the kennel. History says 70% of dogs get adopted fast (3 days), 30% take the slow route (10 days). How long should you expect this pup to wait? Not 3 (not every dog is lucky), not 10 (not every dog is stuck) — the weighted blend.
E[days] = 0.7×3 + 0.3×10 = 2.1 + 3.0 = 5.1 days // verified vs numpy

You may never see 5.1 on the calendar (it's always 3 or 10) — but it's the honest long-run average. The 70/30 are the weights; the days are the values.

What-if table — same dog, different luck
p(fast) / p(slow)E[days] = p·3 + (1−p)·10meaning
90 / 100.9·3 + 0.1·10 = 3.70mostly quick adopters
70 / 300.7·3 + 0.3·10 = 5.10our base case
50 / 500.5·3 + 0.5·10 = 6.50coin-flip kennel
30 / 700.3·3 + 0.7·10 = 7.90hard-to-place dogs

Notice: the EV never leaves the 3→10 window, and it slides toward whichever outcome carries more weight. That's all expected value is.

Step 2 · Expected Value
Second example + the formula (they're the same thing)
🎰 The betting table. A slot machine pays ₹1000 with probability 1%, and ₹0 the other 99%. Worth per pull? Not ₹1000, not ₹0 — the weighted blend: 0.01×1000 + 0.99×0 = ₹10. So a ₹50 spin is a ₹40/pull donation to the casino. 😄
E[X] = Σ p(outcome) × value(outcome) // walk every outcome, weight, sum

The dog's days and the slot's rupees are just different "value" columns fed into the same loop. That's why one idea covers both.

🔧 C-programmer's view — you'd write this in your sleep:
float E = 0;
for (int i = 0; i < n; i++)
    E += p[i] * value[i];   // weight each value by its chance
What-if table — the probability × payout trade-off
p(win)payoutE = p·payoutlesson
1%₹1000₹10base case
5%₹1000₹505× the odds → 5× the value
1%₹5000₹505× the prize → 5× the value (same!)
50%₹20₹10tiny prize, huge odds = same ₹10

Rows 2 & 3 land on the same ₹50 — expected value can't tell a rare-big prize from a common-small one if they multiply out equal. Hold that thought; entropy will care about the difference.

Step 3 · Surprise
Surprise = how big a headline the outcome deserves 📰
🗞️ Two news tickers. "Sun rose this morning" — happens every day, p≈1 → nobody cares, zero surprise. "It rained frogs downtown" — p≈tiny → FRONT PAGE. Surprise = newsworthiness: the rarer the event, the bigger the shock.

So surprise must go up as probability goes down. First instinct is 1/p — but it fails one test: a certain event (p=1) must have zero surprise, and 1/1 = 1, not 0. ❌

surprise(outcome) = log₂(1/p) = −log₂(p) // log₂(1)=0 → certain = 0 surprise ✓

The − is only there because log of a fraction is negative; it flips surprise back positive. No magic. Base 2 → unit = "bits" = how many yes/no questions to pin the outcome down.

What-if table — surprise at different rarities
psurprise = −log₂(p)headline size
1.0 (certain)0.00 bitsnot news
0.50 (coin)1.00 bitmild
0.252.00 bitsnotable
0.10 (rare)3.32 bitsbig shock
0.01 (very rare)6.64 bits🚨 front page
🧠 Deep reason for log (bonus): two independent surprises should add. Probabilities multiply (p₁·p₂), and log is the one function that turns multiply into add. That additivity is why nature picks log — but "certain = 0" is enough to trust it.
Step 4 · Entropy ★★★ core
Entropy = the expected surprise of a pull (the suspense meter 🥁)

Now snap the two halves together: run the expected-value loop, but the "value" of each outcome is its surprise. Entropy is one number for "how mixed / how uncertain is this pile?"

H = Σ p × surprise = Σ p × (−log₂ p) = −Σ p·log₂ p └─────the EV loop─────┘ └─Step-3 surprise─┘
🏏 Two cricket matches. India vs a gully team — result known before the toss, zero suspense, nobody watches → H=0. World-Cup final, scores level, last ball → max suspense, a nation holds its breath → H=max. Entropy measures the nail-biter-ness, not who wins.
🔧 Your bins of flash blocks (● good, ○ bad). Pull one blind — how surprised are you?
The 90/10 twist — worked out

Gut says "90/10 is basically pure → needle near 0." Watch the math disagree:

outcomepsurprise −log₂pcontribution p×surprise
good ●0.90.1520.9×0.152 = 0.137
bad ○0.13.3220.1×3.322 = 0.332 ←!
H = 0.137 + 0.332 =0.469 bits
🪤 The trap (gold). The rare bad block contributes 0.332 — more than double the common good block's 0.137 — even though it's 9× rarer. Rare = shocking, so the 10% minority dominates the entropy. 90/10 isn't ≈0; it's 0.47, nearly half of max. Your gut wanted 0 — that gap is the lesson.

The curve is symmetric (a 90/10 bin is as mixed as a 10/90 bin), 0 at both edges, peak 1.0 at 0.5, and steep near the edges — which is exactly why the rare block moved H so much.

▶ numpy twin — paste into Jupyter, predict the shape first 
import numpy as np, matplotlib.pyplot as plt
p = np.linspace(0.001, 0.999, 500)
H = -p*np.log2(p) - (1-p)*np.log2(1-p)
plt.figure(figsize=(8,5)); plt.plot(p, H, lw=2, color='#6C5CE7')
for prob,name,c in [(1.0,'A pure','green'),(0.5,'B 50/50','crimson'),(0.9,'C 90/10','orange')]:
    h = 0.0 if prob in (0.0,1.0) else -prob*np.log2(prob)-(1-prob)*np.log2(1-prob)
    plt.scatter([prob],[h],color=c,s=90,zorder=5)
    plt.annotate(f'{name}\nH={h:.2f}',(prob,h),textcoords='offset points',xytext=(8,-4))
plt.title('Entropy vs p(good) — the ∩-shaped suspense curve')
plt.xlabel('p(good)'); plt.ylabel('H (bits)'); plt.grid(alpha=.3); plt.tight_layout(); plt.show()
# prints: H(0.9)=0.469  H(0.5)=1.000  H(1.0)=0.000
Step 5 · Information Gain ★★★ core
Information gain = how much a question killed the confusion

A decision tree's whole job: pick the question that best splits the data. Information gain scores each candidate question as the drop in entropy it produces.

Information Gain = H(before) − H(after, weighted) = mess before − mess after
🎮 20 Questions. "Is it bigger than a fridge?" slices the possibilities in half — huge progress. "Does it have a name?" — everything does, you learned nothing. Info gain scores your questions: big drop = great question, zero drop = wasted turn.

Parent bin: 4 good, 4 bad → total confusion, H = 1.0. Try three different questions:

What-if table — same 4●/4○ parent, three questions
splitchildrenweighted H afterIG = 1.0 − afterverdict
Perfect[4/0] & [0/4]0.0001.000🏆 tree picks this
Decent[3/1] & [1/3]0.8110.189weak, but helps
Useless[2/2] & [2/2]1.0000.000❌ learned nothing

The tree computes IG for every candidate question and greedily keeps the biggest. That greedy "maximize the entropy drop" is how a tree grows, node by node. 🌳

🎯 Your thermal project: the throttle-predictor tree asks which telemetry channel best separates "will throttle" from "won't" — temp-slope? write-rate? ambient? — and splits on the biggest info gain. s4 ensembles = many such trees stacked (random forest / XGBoost) = the forecaster family in your research papers. Entropy → info gain → trees → your R2/R3 predictor.
Step 6 · Land it
Decision boundary + the formula sheet 📜

✅ Reach for entropy / information gain when you're measuring the mixedness of a categorical pile or scoring a split / classifier confidence — decision trees, random forests, XGBoost, and (as cross-entropy) the loss of every neural net.

🚫 Where it quietly burns you:
  • Continuous targets (predicting a temperature number, not a class) — entropy is for categories; use variance/MSE reduction instead. Your R2 forecaster (regression) uses MSE; your throttle classifier uses entropy/cross-entropy. Know which brick you're building.
  • Tiny leaves / high-cardinality IDs — info gain is biased toward splits that shatter data into many pure specks (it "loves" a unique-ID column). That's overfitting; gain-ratio / Gini temper it.
Formula sheet
Expected value E[X] = Σ p·value (weighted average) Surprise s(x) = −log₂(p) (bits; certain→0, rare→big) Entropy H = −Σ p·log₂p (expected surprise; 0…1 bit for 2 classes) Information gain IG = H(before) − H(after, weighted)
Key takeaways
  • Expected value, surprise, entropy = one weighted-average loop, three "value" columns.
  • Rare events carry outsized surprise → the 10% minority set the 90/10 bin's entropy to 0.47.
  • Entropy curve: ∩-shape, 0 at edges, peak 1.0 at 50/50, steep near edges.
  • A tree splits by maximizing information gain = biggest entropy drop. Ensembles stack many trees.
  • Entropy is for classes; for a number target use variance/MSE.
🧵 The thread forward: the same −log becomes cross-entropy — the loss every neural net (and every on-device QLoRA fine-tune) trains against. Today's suspense meter is tomorrow's training signal. That's why this is ★★★.
