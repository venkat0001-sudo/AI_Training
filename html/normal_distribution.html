Your SSD controller is sitting at true temp = 80 °C. You poll the sensor 1000 times. You don't get 1000 identical values — you get a bell-shaped scatter around that truth. That scatter IS the normal distribution. Understand it, and anomaly detection is free.

🏋️ Normal Distribution — In The Gym

Most lifters pull the middle weight plates. Almost nobody reaches for the 5 kg or the 125 kg monster stack. μ = the crowd's favourite plate. σ = how adventurous your gym is (tight σ → everyone on the 20 kg; wide σ → chaos from 5 to 80).

📜 The Jutsu Scroll — The Formula
f(x)  =  1 / (σ √2π)  ×  e−(x − μ)² / 2σ²
"The probability density at temperature x, given a sensor with mean μ and noise σ"
↑ Visual twin: drag the μ slider above and watch the whole bell+dots slide left/right. Drag σ slider and watch the bell widen/flatten and the dots spread/tighten in the What-If Panel.
μ (mu) — Mean
The bell's center. The true temperature you're targeting. Most readings land here.
σ (sigma) — Std Dev
How spread the readings are. Tight σ = precise sensor. Wide σ = noisy/aging sensor.
σ² — Variance
σ squared. Raw spread before sqrt. Mean of squared deviations from μ.
e−(x−μ)²/2σ²
Exponential decay away from center. This single term creates the bell shape.
1 / (σ√2π)
Normalizing constant. Ensures area under the full bell = 1.0 (100% probability).
z = (x − μ) / σ
The z-score: how many σ away is this reading? Your anomaly detector is just: |z| > 3.
⚙️ The C-Programmer's View
Python / NumPy
import numpy as np

readings = np.random.normal(80.0, 0.8, 1000)
mu = np.mean(readings) # μ
sigma = np.std(readings) # σ

# z-score anomaly check
z = (readings - mu) / sigma
anomalies = readings[np.abs(z) > 3]
C — how you'd write it in firmware
/* Compute μ and σ from N samples */
float mu = 0.0f, var = 0.0f;
for(int i=0; i<N; i++)
  mu += samples[i] / N;
for(int i=0; i<N; i++) {
  float d = samples[i] - mu;
  var += (d*d) / N;
}
float sigma = sqrtf(var);

/* Anomaly gate */
float z = (x - mu) / sigma;
if(fabsf(z) > 3.0f) fire_throttle();
🔴 Bijuu Detection System — Anomaly Detection in Action
In Sage Mode, Naruto can sense chakra disturbances before the enemy is visible — he doesn't wait to get hit. Anomaly detection does the same: it uses the normal distribution as a baseline, and flags any reading that deviates too far as a threat. That's your proactive throttle trigger.
Decision rule (z-score gate):
z = (x − μ) / σ
|z| < 2  → Normal ✅  (just noise)
2 ≤ |z| < 3 → Warning ⚠️  (watch it)
|z| ≥ 3   → ANOMALY 🔴 → Throttle!
