The spine in one line: direction vs magnitude → dot product mixes them → cosine isolates direction. Everything below is that sentence, unpacked.

1 · A vector has two separable properties

An arrow carries direction (which way it points) and magnitude (how long it is). They're independent — you can change one without the other.

Predict: if I keep the arrow pointing the same way but make it twice as long, what changed — direction, magnitude, or both? Only magnitude. Same heading, longer arrow.

Firmware echo: a signal's shape/heading vs its amplitude. Same waveform, crank the gain → magnitude up, direction unchanged.

2 · Magnitude = length of the arrow

Square the parts, add, square-root. It's Pythagoras stretched to N numbers.

|v| = sqrt(v1² + v2² + ... + vn²) # np.linalg.norm(v)

Walk 3 east, 4 north. Predict the straight-line distance from start: sqrt(3² + 4²) = sqrt(25) = 5

Same move everywhere: TV diagonal (3w×4h → 5), drone speed (6 + 8 → 10 m/s).

3 · Dot product — alignment, but blind to size

A·B = a1·b1 + a2·b2 + ... + an·bn # A @ B

One number measuring how much two vectors align — but it's contaminated by magnitude.

Predict for A=[5,4], B=[10,8]: 5×10 + 4×8 = 50 + 32 = 82

The trap, in two faces:

  • Same direction, different size → wildly different scores: [1,1]·[1,1]=2 but [10,10]·[10,10]=200 — same alignment, 100× score, purely louder
  • Different directions → can give the same score: [1,0]·[5,0]=5 (identical heading) and [5,5]·[1,0]=5 (45° apart) — dot can't tell them apart

4 · Cosine similarity — chop off length, keep heading

cosine(A,B) = (A·B) / (|A| × |B|) # divide out BOTH magnitudes

Divide the dot product by both lengths → the gain cancels, only direction survives. Range −1 … +1.

Predict: two arrows pointing the exact same way (one just longer) — cosine = ? 1.0 — identical direction, length divided out

6 · Three walkers — where dot product gets fooled

A = [5, 4] short stroll, east-northeast D = [50, 40] SAME bearing as A — just hiked 10× farther C = [0, 120] walked due NORTH — different bearing, but very far

D copied A's heading; C went a totally different way. So D is the true match.

Dot product (predict each):

A·D = (5×50)+(4×40) = 410 A·C = (5×0)+(4×120) = 480

So dot says A·C (480) > A·D (410) → "C is the better match!" WRONG — fooled because C walked so far.

Cosine (predict each):

|A|=6.40 |D|=64.0 |C|=120 cos(A,D) = 410/(6.40×64.0) = 1.00 ✓ same bearing cos(A,C) = 480/(6.40×120) = 0.625 ✓ 51° apart

Cosine ranks D (1.0) ≫ C (0.625) — correct. Threw away "how far", kept "which way".

Watch the verdict flip (bars animate):

Cosine similarity is the cosine of the angle between headings. 0°→1, 51°→0.625, 90°→0, 180°→−1.

8 · The decision boundary

When is dot product enough? Only when all vectors are roughly the same length — then magnitude can't distort the ranking (and it's cheaper).

When must you use cosine? The moment magnitudes vary — which in real data is always. Otherwise "rates a lot / walks far" masquerades as "similar".

Firmware echo: comparing two signals' raw correlation without normalizing — a high-amplitude noisy channel out-scores a clean low-amplitude match. Normalize first, then compare.

9 · The 9-rung recall ladder

Try to expand each from memory before peeking at the doc.

  1. vector = direction + magnitude
  2. magnitude = sqrt(sum of squares) = norm
  3. dot = Σ aᵢbᵢ — alignment, but size-contaminated
  4. the trap: same-dir/diff-size, or diff-dir/same-score
  5. cosine = dot / (|A||B|) — direction only
  6. cosine = cos(angle): 0°→1, 90°→0, 180°→−1
  7. far-but-wrong beats near-but-right on raw dot
  8. dot ok if equal lengths; else cosine
  9. NumPy below ↓
import numpy as np a = np.array([5, 4, 1]); b = np.array([10, 8, 2]) a @ b # dot product np.linalg.norm(a) # magnitude (a @ b) / (np.linalg.norm(a)*np.linalg.norm(b)) # cosine
